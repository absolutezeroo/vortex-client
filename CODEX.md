# CODEX for Vortex Client

Use this contract before editing `vortex-client`.

## Objective and scope
- Focus on actionable client tasks with one clear goal per run.
- Prefer minimal diffs over redesign.

## Workflow
1. Read `AGENTS.md` + `CONTEXT.md` + `README.md`.
2. Read target files only.
3. Build in one patch.
4. Keep scope limited to requested domain (onboarding, login, localization, config).
4. Report exact changed files and why.

## Output expectations
- Short rationale first.
- File-by-file summary.
- Explicit assumptions or risks.
- Validation steps and result.

## Collaboration defaults
- If a required choice is unclear, ask one concise question.
- Do not over-document; keep recommendations practical and short.
